# Reference Script: OpenClaw → Claude Code Loop Feature

This is the quality benchmark. Scripts should match this tone, structure, and energy.

## What makes it work

1. **Personal experience framing** — "I will never use X again" not "you should stop using X"
2. **Specific tool names and details** — OpenClaw, Opus 4.6, Claude Max plan, Telegram, Discord
3. **Real frustrations with specifics** — "all the gateway errors that I have to go to my computer and restart the gateway"
4. **Natural speech connectors** — "which I already paid for", "literally last night", "on top of that"
5. **Comparison grounds the value** — doesn't just say "this is great", says "this replaces X and does it better"
6. **No listicle energy** — continuous narrative flow, not "tip 1, tip 2, tip 3"
7. **Soft conditional CTA** — "If you want a full video breakdown, as well as all of my personal templates... click the link in bio"
8. **No SSML breaks needed** — natural speech flows without forced pauses

## The script

I will never use OpenClaw again because Claude Code now has all of the features that OpenClaw has. It just does them better. On top of that, I can now actually use my Claude Max plan, which I already paid for to get access to the Opus 4.6 model, which is by far the best model for agentic assistance. Now, instead of having to worry about all the security issues with OpenClaw, all of the gateway errors that I have to go to my computer and restart the gateway, and all the agent-to-agent messaging errors, I can now set it up all custom with Claude Code hooks with the new Claude Code cron and looping features. It now also allows me to use my entire Claude Code configuration, which I've developed over the course of a better part of a year, through my phone via Telegram, just like I would OpenClaw.
